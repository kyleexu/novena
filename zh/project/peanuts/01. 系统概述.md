# 一、项目背景

**Peanuts** 是一个基于微服务架构的加密货币撮合交易系统，核心目标是用极低延迟的进程间通信（IPC）代替传统的网络 RPC，实现高性能的订单撮合与市场数据分发。

整个系统使用 **Aeron IPC** 作为服务间消息总线，将多个独立的 Spring Boot 微服务串联起来，构成完整的下单 → 锁仓 → 撮合 → 结算 → 行情推送全链路。

```
            Market Service  Market Service  Order Gateway
              (8082/WS)     (OrderBook)    (ExecutionReport)
                   │
                   ▼
             WebSocket Clients
```

# 四、Aeron Stream 对照表

| Stream ID | 发送方 | 接收方 | 消息类型 | 说明 |
|-----------|--------|--------|----------|------|
| **2001** | order | match | OrderProto | 订单入站 |
| **2002** | match | order | ExecutionReportProto | 执行回报 |
| **2003** | match | account, market | TradeProto | 成交明细 |
| **2004** | match | market | OrderBookProto | 订单簿快照 |
| **2101** | order | account | LockRequestProto | 资金锁定请求 |
| **2102** | account | order | LockResponseProto | 资金锁定响应 |

> 所有通信均走 `aeron:ipc`，不经过网络栈，延迟在微秒级别。

# 六、启动顺序

### 模式一：完整链路（含 Kafka）

```bash
# 1. 先启动撮合引擎（内嵌 Aeron MediaDriver，其他服务依赖它）
java -jar match/target/match-1.0.0-SNAPSHOT.jar

# 2. 账户服务
java -jar account/target/account-1.0.0-SNAPSHOT.jar

# 3. Kafka 桥接（需要 Kafka 运行在 localhost:9092）
java -jar bridge/target/bridge-1.0.0-SNAPSHOT.jar

# 4. 订单网关（二选一）
java -jar order/target/order-1.0.0-SNAPSHOT.jar

# 5. 行情服务（二选一，两者都用 8080/8082 端口）
java -jar market/target/market-1.0.0-SNAPSHOT.jar
```

> ⚠️ order 和 market 目前共用 8080 端口，不能同时运行。

### 模式二：核心链路（无 Kafka）

```bash
java -jar match/target/match-1.0.0-SNAPSHOT.jar
java -jar account/target/account-1.0.0-SNAPSHOT.jar
java -jar order/target/order-1.0.0-SNAPSHOT.jar
```

# 七、一笔订单的完整生命周期

```
1. Client  → POST /api/orders               (HTTP)
2. Order   → 校验参数、生成 orderId
3. Order   → 发起资金锁定请求               (Stream 2101)
4. Account → 检查余额、冻结资金
5. Account → 返回锁定结果                   (Stream 2102)
6. Order   → 异步派发订单到撮合             (Stream 2001)
7. Order   → 立即返回 202 Accepted
8. Match   → 接收订单，执行撮合算法
9. Match   → 发布执行回报                   (Stream 2002)
10. Match  → 发布成交记录                   (Stream 2003)
11. Match  → 发布订单簿快照                 (Stream 2004)
12. Account→ 接收成交，完成资金结算         (Stream 2003)
13. Market → 接收成交和订单簿，推送 WebSocket
```

# 八、上线前最小检查清单

1. `mvn -q compile` — 编译通过
2. `mvn -q test -DskipITs` — 单元测试通过
3. 各服务日志中出现 `publisher ready` / `subscriber ready`
4. channel 和 streamId 与约定一致
5. 业务烟测：下单后观察 锁定请求/响应 → 入撮合 → 成交 → 账户结算 → 行情更新
