# 一、职责概述

Market Service（`market` 模块，端口 8082）负责：

1. 订阅撮合引擎产生的成交记录（Stream 2003）和订单簿快照（Stream 2004）
2. 实时计算 Ticker、K 线（OHLCV）等行情指标
3. 通过 **WebSocket** 向客户端推送实时行情数据

Market 服务没有独立的存储，所有数据均保存在内存中，重启后行情历史会丢失。

---

# 二、数据流

```
Match Engine
    │
    ├── Stream 2003 (TradeProto)    → TradeSubscriber → TradeService
    │                                                        ├── TickerSnapshot 更新
    │                                                        ├── CandleSnapshot 更新（1m/5m/15m/1h/1d）
    │                                                        └── WebSocketBroadcaster.broadcastTrade()
    │
    └── Stream 2004 (OrderBookProto) → OrderBookSubscriber → OrderBookService
                                                                 ├── OrderBookSnapshot 聚合
                                                                 └── WebSocketBroadcaster.broadcastOrderBook()
```

# 三、WebSocket 接口

### 3.1 连接地址

```
ws://localhost:8082/ws/market
```

客户端连接后，服务端自动订阅所有默认 Topic：`ticker`、`candle`、`orderbook`、`trade`。

### 3.2 客户端命令（订阅/取消订阅）

```json
// 订阅
{
  "op": "subscribe",
  "topics": ["ticker", "trade", "orderbook.5"]
}

// 取消订阅
{
  "op": "unsubscribe",
  "topics": ["candle"]
}
```

### 3.3 服务端推送消息格式

所有推送消息共享统一信封：

```json
{
  "type": "<ticker|candle|orderBook|trade>",
  "timestamp": 1774541400065,
  "...": "具体 payload"
}
```

# 四、行情数据类型

### 4.1 Ticker（实时成交快照）

```json
{
  "type": "ticker",
  "timestamp": 1774541400065,
  "ticker": {
    "contract": "ETH_USDT",
    "lastPrice": 2973.6308,
    "highPrice": 3002.1132,
    "lowPrice": 2948.1021,
    "volume": 560.0000,
    "turnover": 1664578.1234,
    "tradeCount": 1234,
    "lastUpdateTs": 1774541400065
  }
}
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| lastPrice | 最新成交价 |
| highPrice | 滚动周期最高价 |
| lowPrice | 滚动周期最低价 |
| volume | 成交量（基础货币数量） |
| turnover | 成交额（计价货币金额） |
| tradeCount | 成交笔数 |

### 4.2 Candle（K 线）

```json
{
  "type": "candle",
  "timestamp": 1774541400065,
  "candle": {
    "contract": "ETH_USDT",
    "interval": "1m",
    "openTime": 1774541340000,
    "closeTime": 1774541399999,
    "open": 2975.0000,
    "high": 2978.2000,
    "low": 2970.8000,
    "close": 2973.6308,
    "volume": 15.0000,
    "turnover": 44604.4620,
    "tradeCount": 15
  }
}
```

**支持的 K 线周期：**

| 周期 | 说明 |
|------|------|
| `1m` | 1 分钟 |
| `5m` | 5 分钟 |
| `15m` | 15 分钟 |
| `1h` | 1 小时 |
| `1d` | 1 天 |

### 4.3 OrderBook（订单簿深度）

```json
{
  "type": "orderBook",
  "timestamp": 1774541400065,
  "orderBook": {
    "contract": "ETH_USDT",
    "multiplier": 5,
    "levelStep": 0.5000,
    "bids": [
      { "price": 2973.5000, "quantity": 8.0000 }
    ],
    "asks": [
      { "price": 2974.0000, "quantity": 6.0000 }
    ]
  }
}
```

**订单簿聚合档位：**

通过 `orderbook.<multiplier>` 订阅不同精度的深度数据：

| Topic | multiplier | levelStep（以 ETH_USDT tickSize=0.01 为例） |
|-------|------------|---------------------------------------------|
| `orderbook.1` | 1 | 0.01 |
| `orderbook.5` | 5 | 0.05 |
| `orderbook.10` | 10 | 0.10 |
| `orderbook.50` | 50 | 0.50 |
| `orderbook.100` | 100 | 1.00 |

`levelStep = multiplier × contract.tickSize`

### 4.4 Trade（最新成交）

```json
{
  "type": "trade",
  "timestamp": 1774541400065,
  "trade": {
    "contract": "ETH_USDT",
    "price": 2973.6308,
    "quantity": 1.0000,
    "timestamp": 1774541400065
  }
}
```

# 五、K 线计算原理

K 线由每笔成交记录增量更新，以 1m K 线为例：

```
每笔成交到来时：
1. 根据 trade.timestamp 确定所属的分钟区间 [openTime, closeTime]
2. 若是该区间第一笔成交：open = close = high = low = price
3. 否则：
   - close = price（最新价）
   - high = max(high, price)
   - low = min(low, price)
   - volume += quantity
   - turnover += price × quantity
   - tradeCount += 1
4. 广播最新 CandleSnapshot
```

# 六、客户端断线重连建议

- 采用指数退避策略：1s → 2s → 5s → 30s
- 重连后需重新发送 `subscribe` 命令（订阅状态**不**跨连接保持）
- 服务端发送失败会自动移除该连接

# 七、最小客户端示例（JavaScript）

```javascript
const ws = new WebSocket("ws://localhost:8082/ws/market");

ws.onopen = () => {
  ws.send(JSON.stringify({
    op: "subscribe",
    topics: ["trade", "orderbook.1", "candle.1m"]
  }));
};

ws.onmessage = (evt) => {
  const msg = JSON.parse(evt.data);
  console.log(msg.type, msg);
};

ws.onclose = () => {
  // 实现重连逻辑
};
```
